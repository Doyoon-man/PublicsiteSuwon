# [PRD] 공공기관 웹사이트 입력 편의성 및 접근성 개선 프로젝트

## 1. 프로젝트 배경
현재 수원도시공사를 비롯한 공공기관 웹사이트는 다양한 연령층과 신체적 조건을 가진 시민들이 이용하고 있습니다. 그러나 데스크탑 환경에서 **민감한 호버링 인터랙션**과 **협소한 입력창 디자인**이 고령자 및 운동장애 사용자의 서비스 이탈을 초래하고 있습니다. 본 프로젝트는 이러한 '작지만 치명적인' UI 결함을 해결하여 **디지털 소외 없는 보편적 복지 서비스**를 구현하는 것을 목적으로 합니다.

---

## 2. 문제 정의
* **물리적 장벽:** 입력창 높이가 약 30px로 협소하여 손떨림이 있는 사용자의 마우스 클릭 정확도가 현저히 떨어짐.
* **시각적/흐름적 방해:** 로그인 버튼으로 가는 경로에 위치한 메뉴가 마우스 호버(Hover)만으로 펼쳐져 화면을 가림으로써 사용자의 주 작업(Task)을 방해함.
* **심리적 좌절:** 반복되는 조작 실수로 인해 고령 사용자가 자책감을 느끼고 디지털 서비스를 포기함.

---

## 3. 핵심 UX 가설
> **"만약 우리가** 입력창 높이를 48px 이상으로 키우고, 메뉴 펼침 방식을 '호버'에서 '클릭'으로 변경하면,
> **사용자들은** 의도치 않은 화면 가림 없이 안정적으로 정보를 입력하여 **로그인 및 민원 신청 성공률이 높아질 것이다.**
> **왜냐하면** 조작 허용 범위가 넓어지고 시스템 반응이 예측 가능해져 신체적 제약이 있는 사용자도 정교한 컨트롤에 대한 부담이 줄어들기 때문이다."

---

## 4. 타겟 페르소나
* **성명:** 박철수 (72세, 은퇴 교사)
* **특징:** 지역 커뮤니티 활동을 위해 공공사이트 이용이 잦으나 가벼운 **수전증(손떨림)**이 있음.
* **주요 고통(Pain Point):** 로그인 하려다 메뉴창이 튀어나와 화면을 가릴 때 공포를 느끼며, 작은 입력창을 조준하는 것에 심한 피로감을 느끼고 결국 직접 방문을 선택함.

---

## 5. 핵심 요구사항 및 기능 명세

### 5.1. 입력 필드(Input Field) 개선
| ID | 요구사항 | 상세 설명 | 우선순위 |
| :--- | :--- | :--- | :---: |
| **FR-01** | 입력창 높이 확대 | 모든 ID/PW 및 서식 입력창 높이를 **최소 48px**로 고정함. | P0 |
| **FR-02** | 레이블 고정 배치 | 입력창 내부(Placeholder)가 아닌 **상단/좌측 외부**에 텍스트 레이블을 상시 노출함. | P0 |
| **FR-03** | 클릭 영역 확장 | 입력창 본체뿐만 아니라 주변 마진(Padding) 영역도 클릭 시 포커스가 잡히도록 설계함. | P1 |

## 6. 성공 측정 지표 (Success Metrics)
1. **성공률(Completion Rate):** 로그인 및 민원 신청 페이지의 최종 도달 완료율 **25% 상승**.
2. **정확도(Error Rate):** 입력 과정에서의 오클릭 및 조작 실수 횟수 평균 **50% 감소**.
3. **효율성(Time on Task):** 페이지 진입 후 로그인 완료까지 걸리는 평균 시간 **30% 단축**.
4. **만족도(CSAT):** 고령층 사용자 대상 테스트 시 "조작이 쉽고 안정적이다"라는 긍정 응답 **80% 이상**.

---

## 7. To-Be Task Flow (성공 시나리오)
1. **[로그인 시도]** 메인 화면에서 로그인 버튼으로 마우스 이동 (메뉴 가림 현상 없음).
2. **[입력창 선택]** 48px의 넓은 입력창을 여유롭게 클릭하여 포커스 활성화.
3. **[정보 입력]** 외부에 노출된 명확한 레이블을 확인하며 아이디/비밀번호 기입.
4. **[성공 및 성취]** 단 한 번의 시도로 로그인 성공 후 원하는 민원 신청 서비스 진입.
